# Contributing to Orbit

## 1. Core Philosophy
- **Design First:** Every UI change must meet strict aesthetic standards. No misaligned pixels, no generic default colors.
- **Developer Experience:** Tools must be intuitive, zero-config where possible, and blazing fast.

## 2. Development Workflow
1. **Fork & Branch:** Create a feature branch (`feature/your-feature` or `bugfix/issue-name`) from `main`.
2. **Commit:** Write clear, concise commit messages.
3. **Test:** Verify your changes do not break existing functionality.
4. **Push & PR:** Submit a Pull Request and fill out the provided template.

## 3. Pull Request Guidelines
- Keep PRs strictly scoped to a single fix or feature.
- Include screenshots or videos for *any* UI/UX changes.
- Ensure your code passes all automated CI checks before requesting a review.
